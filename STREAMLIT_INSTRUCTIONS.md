# Streamlit: как опубликовать проект

В этом проекте добавлен `streamlit_app.py` — минимальный интерфейс для демонстрации модели.

Локально

1. Создайте виртуальное окружение и активируйте его:

```powershell
python -m venv .venv
.\.venv\Scripts\Activate.ps1
```

2. Установите зависимости:

```powershell
pip install -r requirements.txt
```

3. Запустите Streamlit:

```powershell
streamlit run streamlit_app.py
```

Откройте URL из консоли (обычно http://localhost:8501).

Streamlit Cloud (быстрый публичный хостинг)

1. Запушьте репозиторий на GitHub (см. `GITHUB_INSTRUCTIONS.md`).
2. Перейдите в Streamlit Cloud и создайте приложение, указав репозиторий и ветку.
3. Команда запуска: `streamlit run streamlit_app.py`.
4. Убедитесь, что модель (`artifacts/bacteria_classifier.pt`) доступна при запуске на облаке:
   - загрузите модель в GitHub Releases и добавьте шаг скачивания в `setup` или `packages` (если Streamlit Cloud поддерживает), или
   - храните модель в публичном хранилище и модифицируйте `streamlit_app.py`, чтобы скачивать файл при старте, если его нет.

Примечания

- `streamlit_app.py` использует ту же логику загрузки модели и предобработки изображения, что и веб-приложение в `src/web`.
- Так как модель часто большой файл, рекомендуется не включать её в git-репозиторий, а хранить через Releases/Drive/S3 и скачивать на этапе развёртывания.

Если хотите — могу:
- добавить автоматическое скачивание модели в `streamlit_app.py` из публичной ссылки, или
- помочь подключить репозиторий к Streamlit Cloud и настроить деплой.
